# Seed:减摩擦简化 — 设计文档

日期:2026-07-22
原则:**收起来,但都留着**。功能一个不删,把"选哪个功能"的决定从用户身上拿走;界面默认安静,想要时随时展开。

## 背景诊断

产品主张"AI 不替你想,让人安静地想",但界面是"一墙按钮请你选":认知页 5 张"点我生成"的 AI 卡竖堆;线程页 7 个面板(多数时候大半是空的);顶栏 7 个导航入口;全站 24 个交互组件。这是 PKM 调研里 "tool obsession over thinking" 的翻版——复杂度不在文件夹,在 AI 功能自助餐台。

## 一、认知页(/mind):从"5 个按钮"到"一进来就有"

- **自动画像**:进页面直接显示今天的认知画像,无需点击。新表 `DailyProfile { userId, date, json }`,`@@unique([userId,date])`,当天首次进页生成并缓存(与每日简报同模式),之后秒开;右上角小「↻ 重新生成」。AI 失败/数据太少时返回的空画像**不缓存**(避免把一次失败缓存一整天)。
- **其余四卡折叠**:思想编年史 / 长期主线 / 自相矛盾 / 认知镜子 收进一行小字入口「还想看:…」,点谁原地展开谁(组件代码不动,默认收起)。
- **静态区紧凑化**:成长曲线、作者占比、分类计数等免费即时数据保留,压缩间距与标题字重,让画像成为视觉焦点。
- 旧 ProfileCard(点击生成式)删除,由服务端直渲染 + RefreshProfile 客户端按钮替代。

## 二、线程页:只显示此刻有意义的

新增通用 `Collapsible`(客户端,label + defaultOpen + children,收起时是一行安静的小字按钮):

| 面板 | 默认 |
|---|---|
| ⚡ 提交前检查 | 有判断(claim)时展开,否则收起 |
| ⚔️ Active Seed 挑战 | 有挑战卡时展开,否则收起 |
| 📜 思想谱系 | 有版本时展开,否则收起 |
| 📝 决策 Memo | 收起 |
| ✍️ 写成一篇 | 收起 |

打开一条线看到的是这条线本身,不是七个工具箱。

## 三、导航 7 → 4

- 常驻顶栏:**星图 · 简报 · 认知 · 设置**
- **阅读并进简报**:简报底部加「推荐阅读」小节(最多 3 条 + 「全部 →」链到 /reading;/reading 页保留可达)
- **导入/产出**:设置页把过时的"下一步 Slice2+"清单换成「工具箱」小节,链到 /import、/outputs、/reading(页面全保留,只是不占顶栏)

## 不做

- 不删任何功能/页面/AI 能力;不改捕获与星图(上一轮刚做过);不动 Telegram bot。
