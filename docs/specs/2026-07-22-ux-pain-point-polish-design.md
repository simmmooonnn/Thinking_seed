# Seed:基于真实用户痛点的体验打磨 — 设计文档

日期:2026-07-22
输入:三份联网调研(捕获摩擦 / 笔记坟场 / PKM 与 AI 笔记工具信任),约 40 条带来源的真实用户痛点。
原则:一切以人的体验为第一目的;不加新"功能面",只让已有功能更值得信任、更不打扰、更零摩擦。

---

## 调研结论 → 设计决策(六项)

### 1. 星图防"毛球":专注模式(结构性风险,最高优先级)
**痛点**:图谱视图被骂"好看没用"的根因是**默认展示全部数据的力导向布局在规模变大后必然退化成毛球**(社区阈值 ~200-500 条;Obsidian 论坛、Roam issue #10)。被认可的用法全部带约束:局部图、筛选后、有层级。Seed 把 3D 星图当主界面,风险被放大到第一入口。
**设计**:星图增加两档视图:
- **专注模式(数据多时默认)**:只显示 根 + 思想线程星 + 线程间紫色主线;散点(entry 星尘)默认隐藏,点击某条线程星后只展开**这一条线**的星尘。= "局部图谱"。
- **全部模式**:现在的样子(演讲/鸟瞰用)。
- 切换用星图上的一个轻量分段控件(专注 | 全部);当 entry 数 > 40 默认专注,否则默认全部。图例、搜索、悬停高亮在两档都工作。

### 2. 三卡合一"信使":对抗提醒疲劳
**痛点**:64% 用户在一周收到 5+ 通知后卸载;提醒越多单条越没用;"未处理计数器"直接诱发弃用(愧疚仪式)。Seed 首页现在同时浮 3 张卡(重新浮现/Active Seed/主动提醒)= 教科书式提醒疲劳。
**设计**:合并为**一张**信使卡(底部居中,原 nudge 位置):
- 每次只显示**一条**内容,优先级:挑战(⚔️) > 主动提醒(🌱,AI 一天一条、沿用本地缓存) > 重新浮现(🌙)。
- 「换一条」轮换其余;**不显示任何数量角标**;✕ 一次关掉整张卡(当日不再出现,localStorage 记忆,新内容新 id 次日再来)。
- 删除首页的 ResurfaceCard、ChallengeAlert、ProactiveNudge 三个独立浮层(组件文件删除,逻辑并入 MessengerCard)。

### 3. 重新浮现要有"理由",不要冰冷倒计时
**痛点**:Readwise 式"随机旧内容+时间戳"被评"重复无聊";用户判断值不值得看的依据是"我现在是否在乎",真正促使回看的是**与当下相关**。
**设计**:重新浮现条目服务端计算**浮现理由**:若休眠线程经 ThreadLink 与一条最近 7 天活跃的线程相连 → 理由 =「它和你最近在想的「X」相连」;否则退回「你 N 天没碰它了」。有理由的排前面。卡片同时显示该线程的问题(question),帮助重建语境。纯启发式,零 AI 成本。

### 4. 捕获回执 + 一步纠正:保住 AI 归类的信任
**痛点**:AI 跨条目综合判断准确率明显更低(80%→60%);信任不取决于准确率本身,而取决于**可见、可查、可撤销**;纠正若比手动分类还麻烦,等于把负担从捕获前挪到捕获后。
**设计**:网页捕获(文字+语音)成功后,输入条上方浮一张**8 秒自动消失**的回执小卡:
- 内容:🌱 已记下 · {类别} · 归入「{线程}」/暂放收件箱;若归入了线程且该线有 question:附一行「这条线在问:{question}」(语境连续性,把"回看"嵌进"记录"的瞬间)。
- 一步纠正:卡上放 5 个类别小 chip(点击即 setKind)+「移出线程」;不强制任何确认,不点就 8 秒后消失(零"必须处理"感)。
- 需要 `createEntry` 返回 {id, kind, threadId, threadTitle, question};`/api/voice` 响应里附同样信息。

### 5. 去愧疚化:删掉积压计数器
**痛点**:"N 条待处理"型角标把中性负担变成主动焦虑,用户的反应是关掉整个功能而不是去处理。
**设计**:简报页 Listener/Gardener 移除「N 条待归类」「Inbox N 条散念待安置」「N 条线程逾十日未动」等积压数字;成就型数字保留(近七日记下 N 条)。改为无数字的温和提示(如「有些散念还飘着,想安置的时候去星图看看」)。

### 6. 画像/编年史附"证据链"
**痛点**:"AI 摘要让人不看原文"、AI 参与痕迹本身削弱可信度;唯一解法是结论**可回溯**到原始记录——这正是"AI 不替你想"理念的交互落地。
**设计**:
- 认知画像卡:结果下方加一行 mono「依据」:基于 N 条记录 · M 条思想线 · 第一作者 x% · y 个判断无证据 · z 个假设未收口(来自 computeProfile 已有统计,零额外 AI 成本;computeProfile 返回 {profile, basis})。
- 思想编年史:AI schema 的每个 chapter 增加 `threads: string[]`(该节涉及的思想线名);渲染成小 chip,能匹配到线程 id 的做成链接跳转 `/thread/<id>`。

### 附:桌面捕获快捷键
**痛点**:"打开工具写一行字比丢掉念头更打断心流";flomo 胜在"打开即写"。
**设计**:首页任意处按 `/`(不在输入框内、非 IME 组词时)直接聚焦捕获输入条;Esc 失焦。

## 不做(YAGNI / 有意保留)
- AutoSeed 后台生成挑战**保留**(用户明确喜欢);调研警示"AI 主动挑战有冒犯风险"的应对是**呈现降级**(并入信使卡、可无痕忽略),而非砍生成。
- Telegram 端 inline 按钮纠正、移动端轻量视图、语音低置信度标记:记为后续,不进本轮。
- 不改数据模型、不加新表。
